Two-outcome
KB Security

Malicious KB chunks are silently dropped. The agent answers from whatever clean chunks remain. Only when every chunk is malicious is the request blocked.

OUTCOME A — INJECTION DETECTED

Log a security event and drop that chunk. Continue with the remaining chunks. Only triggered when no clean chunks survive.

OUTCOME B — CLEAN KB

Re-rank chunks by relevance. Answer using only the chunks that passed the scanner. Dropped chunks are never shown to the LLM.
